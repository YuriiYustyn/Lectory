Команди для локального гіту

  • git config --global user.name blabla - задає імя користувача
  • git config --global user.email blabal@bla - задає емейл користувача
  • git init - ініціалізує пустий репозиторій
  • git status - перевіряє статус
  • git commit - робить"зліпок"попередньої версії із змінами
  • git log - відкриває список комітів
  • git checkout - створює нову гілку і переходить на неї
  • git branch - відображає всі гілки
  • git push - відправляє на репозиторій зліпки проекту
  • git pull - отримує зміни з репозиторію
  • git merge - обєднує зміни в одне ціле з різних гілок
  • git clone - клонує репозиторій з історією всіх змін

gitignore файл вказує СКВ які файли не відслідковувати, тобто ігнорувати

  • git revert - команда призначена для безпечної відміни публічних комітів
  • git reset - команда для відміни змін
  • git cherry-pick - використовується щоб, взяти зміни внесені якимось комітом
    і попробувати застосувати їх знову у вигляді нового коміта зверху поточної гілки
  • git rebase - переносить зміни з одної гілки в іншу
  • git stash - команда призначена для того, щоб помістити поточні зміни в окреме місце
    і повертає файл до початкової версії
